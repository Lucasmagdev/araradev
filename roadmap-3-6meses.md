# Roadmap — próximos 3-6 meses

Prioridade definida em 2026-06-12. Revisar mensalmente.

> **Execução prática**: a Prioridade 1 (fundamentos técnicos) virou uma trilha gamificada em `app/` (8 fases, 59 lições — teoria com quiz + exercícios de código com testes automáticos + checklists). Ritmo combinado: ~2-3 lições/dia. Progresso fica salvo no navegador (localStorage). As "Fases" do app substituem a divisão por "Mês" abaixo — use o app como guia do dia a dia, este arquivo como visão geral.

## Prioridade 1: Fundamentos técnicos sem IA (peso ~40%)

Objetivo: conseguir ler, debugar e julgar código gerado sem depender de pedir pra IA resolver de novo.

Plano mês a mês (1 foco por mês, ~3-4h/semana):

- **Mês 1 — Lógica de programação fundamental (do zero, sem IA)**
  - Variáveis, condicionais, loops, funções (parâmetro/retorno) — revisar conceito antes de praticar
  - Foco especial no padrão "acumulador": loop que constrói um resultado (soma, contagem, concatenação)
  - 1 exercício pequeno por dia (somar lista, contar pares, inverter string) — escrever sozinho primeiro, comparar com IA depois
- **Mês 2 — Estruturas de dados básicas**
  - Array/lista (índice, ordem) vs objeto/dicionário (chave→valor) — quando usar cada
  - Exercícios manipulando ambos (filtrar, transformar, percorrer) na linguagem do trabalho
- **Mês 3 — SQL e modelagem de dados**
  - Já tem boa intuição de leitura (#4 do diagnóstico saiu certo) — agora foco em escrever na mão: SELECT com WHERE/JOIN/GROUP BY
  - Modelar do zero um schema simples (ex: cliente-pedido-produto)
- **Mês 4 — Debug e leitura de código gerado por IA**
  - Pegar trecho de código gerado num projeto seu e explicar linha a linha pra si mesmo
  - Exercício de "code kata": resolver problema pequeno sem IA, comparar depois com solução da IA
- **Mês 5 — Testes automatizados**
  - Aprender a escrever testes unitários/integração na stack que usa (escrever o teste, não só pedir pra IA gerar)
- **Mês 6 — Arquitetura leve + segurança básica (OWASP Top 10)**
  - Padrões básicos (camadas, API design) + revisar projetos próprios (Gontijo/Codexy) procurando erros comuns

## Prioridade 2: Inglês — hábito diário, não projeto (peso ~30%)

Meta: 20-30min/dia, todo dia, sem exceção (mais importante que sessões longas esporádicas).

- Trocar idioma do sistema/IDE/Claude Code pra inglês
- Assistir 1 vídeo técnico/devlog em inglês por dia (com legenda em inglês, não PT)

Revisão em 3 meses: avaliar progresso, decidir se precisa de algo mais estruturado (curso, intercâmbio, professor)
